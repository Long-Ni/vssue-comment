Yarn 常用命令

Ni   in 大连
Mon Nov 23 2020
  • Yarn

Yarn 提供了丰富的命令行集合，可帮助了解 Yarn 包的各个方面，包括安装，管理，发布等。

# Yarn 运行脚本

yarn run [script] [<args>]
# 也可以省略 run，直接使用脚本名称运行。
yarn [script] [<args>]

yarn run 用来执行在文件 package.json 的 script 字段中定义的脚本，例：

{
  "name": "vuepress-starter",
  "version": "0.0.1",
  "scripts": {
    "dev": "vuepress dev src",
    "build": "vuepress build src"
  }
}

想要执行 script 字段中 dev 对应的脚本 vuepress dev src ，则使用：

yarn run dev
# 或者
yarn dev

# Yarn 显示包信息

yarn info <package>[@version] [<args>]
# 要输出 json 格式信息
yarn info <package>[@version] [<args>] --json

获取包信息并默认返回树结构。从网络检索包信息，所以非本地包也支持。例：

# 树格式
yarn info vx.x.x
{ name: 'react',
  version: '15.4.0-rc.2',
  description: 'React is a JavaScript library for building user interfaces.',
  time: { modified: '2016-10-06T22:09:27.397Z', ... } ... }
# json 格式
{"type":"inspect","data":{"name":"react","time":{...}}}
{"type":"finished","data":417}

# Yarn 列出项目所有依赖

yarn list [--depth] [--pattern]

列出当前文件夹所有依赖，例：

yarn list vx.x.x
├─ package-1@1.3.3
├─ package-2@5.0.9
│  └─ package-3@^2.1.0
└─ package-3@2.7.0

# Yarn 管理配置文件

设置：yarn config set <key> <value> [-g|--global]

获取：yarn config get <key>

删除：yarn config delete <key>

查看：yarn config list

# Yarn 缓存

Yarn 将每个包存储在文件系统用户目录中的全局缓存中。

输出所有已缓存的包：

yarn cache list [--pattern]

输出全局缓存当前存储地址：

yarn cache dir

清除全局缓存：

# 清除所有包
yarn cache clean
# 指定一个或多个想要清除的包
yarn cache clean [module_name...]

修改缓存地址：

# 通过配置文件修改缓存地址
yarn config set cache-folder <path>
# 通过 --cache-folder 标签指定缓存地址
yarn <command> --cache-folder <path>
# 通过环境变量修改缓存地址
YARN_CACHE_FOLDER=<path> yarn <command>

# Yarn why

显示包为什么被安装的信息。

yarn why <query>
# 关于强制参数 query 可以是以下任意一项：
# · 一个包名。
yarn why jest
# · 一个包文件夹。
yarn why node_modules/once
# · 包文件夹中的一个文件。
yarn why node_modules/once/once.js

该命令将确定为什么安装了软件包，例如详细说明了哪些包依赖于该包，或者是否在 package.json 清单中将其明确标记为依赖项。例：

yarn why vx.x.x
[1/4] 🤔  Why do we have the module "jest"...?
[2/4] 🚚  Initializing dependency graph...
[3/4] 🔍  Finding dependency...
[4/4] 🚡  Calculating file sizes...
info Has been hoisted to "jest"
info This module exists because it's specified in "devDependencies".
info Disk size without dependencies: "1.29kB"
info Disk size with unique dependencies: "101.31kB"
info Disk size with transitive dependencies: "20.35MB"
info Amount of shared dependencies: 125

# Yarn audit

针对已安装的包执行漏洞审查。

yarn audit [--verbose] [--json] [--level] [--groups]
