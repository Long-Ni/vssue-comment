# 1. 创建 Private VPC

创建私有 VPC （Private VPC）。Private VPC 是没有公有子网和互联网网关的 VPC。整个网络不接入 Internet。

Private VPC 特点：

  • 该 VPC 没有互联网网关。
  • 该 VPC 没有 NAT 设备。
  • VPC 中所有流量不接入互联网。

# 2. 创建子网（Subnet）

创建多个子网，通过路由表将不同子网的流量指向不同的对等 VPC。创建两个 VPC 与一个 VPC 中的两个子网对等 (opens new window)型的对等连接。

# 3. 创建 VPC 对等连接（Peering Connection）

VPC 对等连接是两个 VPC 之间的网络连接，可通过此连接不公开地在这两个 VPC 之间路由流量。这两个 VPC 中的实例可以彼此通信，就像它们在同一网络中一样。

# 3.2 关于 VPC Peering Connection

Peering Connection 特点：

  • 建立 Peering Connection 的两个 VPC 中的实例可以彼此通信，就像他们在同一个网络中一样。
  • 使用私有 IP 进行通信
  • 跨 VPC 可以访问的资源包括 EC2 Instance、Amazon RDS数据库和 Lambda 函数。
  • 所有区域间流量都经过加密，没有单点故障或带宽瓶颈。
  • 流量一直处于全球 AWS 骨干网中，不会经过公共 Internet，这样可以减少面临的威胁，例如常见漏洞和 DDos 攻击。
  • 区域间 VPC 对等连接提供了一种在区域间共享资源或为实现地理冗余性而复制数据的简单经济方式。

如果对等关系中的任一 VPC 具有下列连接之一，则无法将对等关系扩展到该连接 (opens new window)：

  • 与企业网络之间的 VPN 连接或 AWS Direct Connect 连接。
  • 通过互联网网关（Internet Gateway）建立的 Internet 连接。
  • 在私有子网中通过 NAT 设备建立的 Internet 连接。
  • AWS 服务的网关终端节点（Endpoint of gateway type）；例如，Amazon S3 的终端节点。
  • （IPv6）ClassicLink 连接。

# 4. 创建 VPC 终端节点（Endpoint）

通过 Endpoint 建立 PrivateLink，以允许 AWS 服务（Elastic Beanstalk，s3，CloudFormation，SQS，CloudWatch...）连接到 Private VPC 中，对其进行操作。使整个网络只存在与 AWS 主干网内，而不接入互联网（internet）。

# 4.1 VPC Endpoint 使用

# 4.1.0 前提

  1. 在其 VPC 内创建两个私有子网。

  2. 启用 VPC 的 DNS 支持。将两项属性设置为 true：DNS Resolution & DNS Hostnames

    步骤：查看和更新针对 VPC 的 DNS 支持 (opens new window)

# 4.1.1 创建与查看 Interface Endpoint

创建步骤：创建 Interface Endpoint (opens new window)

依次为以下 Amazon 服务创建 Interface Endpoint：

  • elasticbeanstalk：（required）服务名称为com.amazonaws.region.elasticbeanstalk。确保 Elastic Beanstalk 与 Private VPC 一起使用。参考将 Elastic Beanstalk 与 VPC 终端节点结合使用 (opens new window)
  • elasticbeanstalk-health：（required）服务名称为com.amazonaws.region.elasticbeanstalk-health。确保 Elastic Beanstalk Enhanced Health 与 Private VPC 一起使用。关于 Elastic Beanstalk Enhanced Health (opens new window)
  • cloudformation：（required）服务名称为com.amazooptionnaws.region.cloudformation。参考设置 AWS CloudFormation 的 VPC 终端节点 (opens new window)。关于 AWS CloudFormation (opens new window)
  • events：（optional）服务名称为com.amazonaws.region.events。确保 CloudWatch Events 与 Private VPC 一起使用。参考将 CloudWatch Events 和接口 VPC 终端节点一起使用 (opens new window)。关于 Amazon CloudWatch Events (opens new window)
  • logs：（required）服务名称为com.amazonaws.region.logs。确保 CloudWatch Logs 与 Private VPC 一起使用。参考将 CloudWatch Logs 和接口 VPC 终端节点一起使用 (opens new window)。关于 Amazon CloudWatch Log (opens new window)
  • monitoring：（optional）服务名称为com.amazonaws.region.monitoring。确保 CloudWatch 与 Private VPC 一起使用。参考为 CloudWatch 创建 VPC 终端节点 (opens new window)。关于 Amazon CloudWatch (opens new window)
  • sqs：（required）服务名称为com.amazonaws.region.sqs。确保 Amazon SQS 与 Private VPC 一起使用。参考Amazon SQS 的 Amazon Virtual Private Cloud 终端节点 (opens new window)。关于 Amazon SQS (opens new window)

创建 Interface Endpoint 时需要重点注意的点：

  1. Enable DNS name：true。环境实例上的 Elastic Beanstalk 代码使用公有终端节点将流量发送到所有 AWS 服务。要确保此流量通过，请在配置所有接口 VPC Endpoint 时选择 Enable DNS name 选项。这会在你的 VPC 中添加一个 DNS 条目（entry），该条目将公有服务终端节点映射到接口 VPC 终端节点。
  2. 选择要在其中创建 Endpoint 的子网。可以在不同的可用区中指定多个子网 (在服务支持的情况下)，以帮助确保你的 Interface Endpoint 能够在出现可用区故障时复原。在此情况下，我们将在你指定的每个子网中创建一个终端节点网络接口。
  3. Security Group：如果在创建 Endpoint 时不指定安全组，则系统默认将 VPC 默认安全组关联到该 Endpoint 上。要使 Endpoint 与实例通信，则启动实例时必须将该默认安全组关联到实例。（默认安全组允许来自分配给相同安全组的网络接口及其关联实例的入站流量，创建 Endpoint 时推荐选择默认安全组）

查看步骤：查看 Interface Endpoint (opens new window)

# 4.2 关于 VPC Endpoint

VPC Endpoint 可通过 AWS privateLink 实现 Private VPC 与支持的 AWS services、VPC endpoint services 之间的私有连接。

AWS PrivateLink 是使你能够通过 private IP 私密访问服务的一项技术。

VPC Endpoint 特点：

  • VPC Endpoint 使你能够将 VPC 私密地连接到支持的 AWS services 和 VPC endpoint services（由 AWS PrivateLink 提供支持），而无需互联网网关、NAT 设备、VPN 连接或 AWS Direct Connect 连接。
  • VPC 中的实例无需公有 IP 地址便可与服务中的资源进行通信。
  • VPC 和其他服务之间的流量不会脱离 Amazon 网络。
  • Endpoint 是虚拟设备。它们是水平扩展、冗余和高度可用的 VPC 组件。

VPC Endpoint 有两种类型：

  1. Interface Endpoint：是一个弹性网络接口，具有来自子网 IP 地址范围的私有 IP 地址，用作发送到受支持的服务的通信入口点。

    • 对于每个接口终端节点，每个可用区您只能选择一个子网。

    • 某些服务支持使用 Endpoint 策略来控制对服务的访问。

    • 一些可用区不支持 Interface Endpoint 服务。

    • 如果子网的网络 ACL 限制流量，您可能无法通过终端节点网络接口发送流量。请确保您增加了相应的规则，允许与子网的 CIDR 块之间的往返流量。

    • 接口终端节点仅支持 TCP 流量。

    • 仅在同一区域内支持 Endpoint。无法在 VPC 和其他区域内的服务之间创建终端节点。

      Private DNS for Interface Endpoint：

      • 创建 Interface Endpoint 时，将会生成可用于与服务通信的端点特定的 DNS 主机名（Endpoint-specific DNS hostnames：vpce-000000000-yyyyyyy.elasticbeanstalk.ap-northeast-1.vpce.amazonaws.com）。

      • 对于 AWS 服务和 AWS Marketplace 合作伙伴服务，私有 DNS 选项（默认启用）会将私有托管区域（private hosted zone）与你的 VPC 相关联。

      • 托管区域包含服务的默认 DNS 名称（Default DNS name：elasticbeanstalk.ap-northeast-1.amazonaws.com）的记录集，用于解析你的 VPC 中的 endpoint network interface 的私有 IP 地址。这使你能够使用服务的默认 DNS 主机名而不是终端节点特定 DNS 主机名向服务发出请求。

      • Endpoint 未启用 private DNS ：任一子网中的实例都可以使用端点特定的 DNS 主机名通过 Interface Endpoint 向 Amazon 服务发送请求。公有子网中的实例可以使用其默认 DNS 名称，通过 AWS 区域中的公有 IP 地址空间与 Amazon 服务通信。

      • Endpoint 启用 private DNS ：任一子网中的实例都可以使用默认的 DSN 主机名或端点特定的 DNS 主机名，通过 Interface Endpoint 向 Amazon 服务发送请求。

      • 在 Private VPC 中使用 Endpoint 必须启用 private DNS。（如果你的 VPC 不是私有的，且具有 Internet 访问权限，如果将 VPC Endpoint 禁用了 Enable DNS name，则流向相应服务的流量将通过 Internet 传输。这可能不是你期望的。使用 Private VPC 可以检测到这个问题，因为它可以防止此流量通过并且返回错误。对于公有 VPC 不会有任何指示。）

        注意：

        要启用 Private DNS，必须将 VPC 的以下属性设置为 true： enableDnsHostnames 和 enableDnsSupport。

  2. Gateway Endpoint：是一个网关，作为路由表中指定路由的目标，用于发往受支持的 AWS 服务的流量。

    • 仅支持 IPv4 流量

    • 仅在同一区域内支持 Endpoint。无法在 VPC 和其他区域内的服务之间创建终端节点

# 5. 使用 EB 在 Private VPC 中创建环境并部署代码

如果在 Private VPC 中创建 Elastic Beanstalk 环境，则该环境无法访问 Internet。你的应用程序可能访问 Elastic Beanstalk 服务或者其他服务，或者实例会将环境运行状况信息发送到增强型运行状况服务（enhanced health service）。

所以要使用 EB 在 Private VPC 中创建环境并部署代码，必须先在其 Private VPC 中配置 VPC Endpoint，即完成上述第 4 点。

    • 对于 Network，点击 Network 卡片上 Edit 按钮，进入 Network 设置页面：
      • Virtual private cloud (VPC)：选择新创建的 Private VPC
      • Load balancer settings： 对于 Visibility 此处必须选择 internal（Private VPC 的情况下），并选择负载均衡子网（子网用于配置负载均衡器，此处至少选择两个子网。由于子网不跨可用区，每个子网处于一个可用区内，所以此处的两个子网必须处于不同的可用区内。参考创建负载均衡器必须至少选择两个可用区子网 (opens new window)）
      • Instance settings：不勾选 Public IP address 选项，并选择实例子网（实例将启动在此时选择的任一子网内。可以选择上述负载均衡器子网，也可以选择其他子网，但实例子网所处可用区与负载均衡器所处可用区必须一致）
    • 对于 Instances，点击 Instances 卡片上 Edit 按钮，进入 Instances 设置页面（要完成此步骤，必须先完成对 Network 的设置。）：
      • EC2 security groups：此处选择安全组时，要确保启动环境过程中能使 EB 启动的 instance 与 Endpoint 进行通信。所以此处选择的安全组与创建 Endpoint 时选择的安全组有关。如果创建 Endpoint 时，为服务指定了 VPC 的默认安全组，那么此处必须选择该 VPC 的默认安全组（推荐使用默认安全组）。如果创建 Endpoint 时，为服务指定了自定义的安全组（该安全组必须允许来自 instance 流量入站），那么此处必须选择允许来自 Endpoint 流量入站的安全组。（由于 AWS bug，此处可能会显示相同的多的安全组，选择符合条件的其中一个或者多个即可，不必重复选择）
    • 对于 Capacity，点击 Capacity 卡片上 Edit 按钮，进入 Capacity 设置页面（要完成此步骤，必须先完成对 Network 的设置。）：
      • Auto Scaling Group：由于在 Presets (预设) 部分选择了 High availability，所以此处 Environment type 值默认为 Load balanced，不能修改。Instances 的 Max 值可按需设置，此处我设置为 1。
    • 对于 Security，点击 Security 卡片上 Edit 按钮，进入 Security 设置页面：
      • Virtual machine permissions：对于 EC2 key pair 选择任一密钥文件。
    • 对于 Managed Updates，点击 Managed Updates 卡片上 Edit 按钮，进入 Managed Updates 设置页面：
      • Managed platform updates：对于 Managed updates 不勾选 Enabled ，取消自动更新平台功能。
    • 其他选项保持默认值或按需设置
  6. 点击 Create environment 按钮，开始创建新环境。

# 5.1.3 部署代码

如果环境正常启动并运行，则可开始部署自己的代码到 EB 中：（此处只涉及使用 EB CLI 进行部署）

  1. 安装 EB CLI，参考安装 EB CLI (opens new window)

  2. 配置 EB CLI，参考配置 EB CLI (opens new window)（需要先进入代码根目录下，再进行该配置）

  3. 部署代码：

    在代码目录下执行以下命令进行部署

    eb deploy environment-name

    注意： 如果你的应用程序具有诸如语言运行时包之类的依赖项，它可能会尝试在环境创建期间从 Internet 下载并安装这些依赖项，但此操作将失败。要避免发生此失败，请在应用程序的源包中包含所有依赖项包。

# 5.1.4 后续操作

  1. 修改安全组

    使用 AWS 的安全工具 Trusted Advisor (opens new window) 检测整个环境是否处于安全状态 (opens new window)。Elastic Beanstalk 在创建环境的过程中分别为 ELB 和 Instance 自动创建了安全组。如下

    ELB security group：（sg-aaaaaaaaa）

    Type Protocol Port range Source
    HTTP TCP 80 0.0.0.0/0

    Instance security group：（sg-bbbbbbbbb）

    Type Protocol Port range Source
    HTTP TCP 80 sg-aaaaaaaaa
    SSH TCP 22 0.0.0.0/0

    修改 Instance security group，以收紧实例访问权限，修改后 security group：（sg-bbbbbbbbb）

    Type Protocol Port range Source
    HTTP TCP 80 sg-aaaaaaaaa

    再次检测环境是否处于安全状态 (opens new window)。

  2. 检查 instance 运行状况并监控 EB 环境 (opens new window)

至此，使用 EB 在 Private VPC 中创建环境并部署代码所有步骤已经完成。

使用 EB 在 Private VPC 中创建环境并部署代码时需要重点注意的点：

  1. 如果只有一台 Instance 也需要创建 ELB。并且将 ELB 指向内部（internal）。如果要为某个环境创建 ELB，则该环境下必须至少有两个 subnet 。
  2. 安全组选择 VPC 默认安全组。（确保 EB 启动的实例可以被 Endpoint 访问到）

# 6. 使 Private VPC 中的实例通过 Peering Connection 连接到对等 VPC 的 RDS 中

要使 Private VPC 中的实例通过 Peering Connection 连接到对等 VPC 的 RDS 中，必须先创建 Peering Connection，即完成上述第 3 点。

完成对等连接创建后，只需修改对等 VPC 中 RDS 安全组，以允许 Private VPC 中的 CIDR 块有权限访问。

在 RDS 安全组中添加如下入站规则：

Type Protocol Port range Source
MYSQL/Aurora TCP 3306 172.16.0.0/25

其中 Source 的值为 Private VPC CIDR 块或者其子网 CIDR 块。
