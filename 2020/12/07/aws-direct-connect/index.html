# AWS Direct Connect 是什么？

通过光缆将内部网络连接到 AWS Direct Connect (opens new window)。

一端连接到内部网络的路由器，另一端连接到 AWS Direct Connect 的路由器。

此连接可以创建虚拟接口。

通过此连接可以创建直接连接到公有 AWS 服务（如 Amazon S3）或 Amazon VPC 的虚拟接口，从而绕过网络路径中的 Internet 服务提供商。

AWS Direct Connect 站点在其关联的区域提供对 AWS 的访问权限。

# 什么时候使用 AWS Direct Connect？

通过 AWS Direct Connect，可以连接到 AWS 区域内的所有AWS资源。

将您的业务关键数据直接从您的数据中心、办公室或主机托管环境中传输到AWS中、或者从AWS中传输数据到您的数据中心、办公室或主机托管环境中。

绕过您的互联网服务提供商并消除网络拥堵。

食用 AWS Direct Connect 的使用案例：

  • Working with Large Data Sets（处理大型数据集）：节省带宽，消除网络拥堵，降低联网成本。
  • Real-time Data Feeds(实时数据反馈)：网络延迟始终保持一致。
  • Hybrid Environments（混合环境）

# AWS Direct Connect 站点

AWS Direct Connect 已经在全球多个站点提供。在某些园区设置中，可通过来自由同一校园内同一提供商运营的其他数据中心的标准交叉连接访问 AWS Direct Connect。下面列出了所有可通过标准交叉连接访问 AWS Direct Connect 的 AWS Direct Connect 站点和园区数据中心的。为了实现高可用性，AWS 建议客户使用多个站点。

客户可以从我们以下任何 AWS Direct Connect 站点（中国除外）访问任何 AWS 区域。下表中的 "关联的 AWS 区域 "列是默认区域，在 AWS 管理控制台中，您将在该区域下找到特定的 AWS Direct Connect 站点。通过 Direct Connect Gateway 和公共虚拟接口，您可以从所选择的站点访问任何其他 AWS 区域。

在 AWS Direct Connect 站点没有设备的客户可以在 AWS 合作伙伴网络成员的协助下访问 AWS Direct Connect。AWS 得 APN(AWS Partner Network) 技术和咨询合作伙伴在不断增加，他们可以协助您访问 AWS Direct Connect 服务。这些 APN 合作伙伴可以帮助您在 AWS Direct Connect 站点与您的数据中心、办公室或主机托管环境之间建立网络电路，或协助您构建混合环境。

refer: https://aws.amazon.com/directconnect/features/

# AWS Direct Connect 的两个组件：

# - Connections

在 AWS Direct Connect 站点创建 Connections 以建立从本地到 AWS 区域的网络连接。

  1. 专有连接
  2. 托管连接

# - 虚拟接口

创建 虚拟接口 以启用对 AWS 服务的访问。公有虚拟接口允许对公有服务的访问，如 Amazon S3。私有虚拟接口允许对你的 VPC 的访问。

# 网络要求

要在 AWS Direct Connect 站点使用 AWS Direct Connect，网络必须符合以下条件：

  • 你的网络必须使用具有适用于 1 GB 以太网的 1000BASE-LX (1310nm) 收发器或适用于 10 GB 以太网的 10GBASE-LR (1310nm) 收发器的单模光纤。
  • 必须禁用端口的自动协商功能。必须手动配置端口速度和全双工模式。
  • 必须跨整个连接 (包括中间设备) 支持 802.1Q VLAN 封装。
  • 您的设备必须支持边界网关协议 (BGP) 和 BGP MD5 认证。

此外必须满足以下条件之一：

  • 您的网络托管于现有的 AWS Direct Connect 站点。
  • 您正与作为 AWS Partner Network (APN) 成员的 AWS Direct Connect 合作伙伴开展合作。
  • 您正与独立的服务供应商合作连接到 AWS Direct Connect。
