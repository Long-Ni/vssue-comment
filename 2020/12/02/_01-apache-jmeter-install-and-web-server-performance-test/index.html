# 1. 安装 Java

要安装 Apache JMeter 首先必须安装 Java。

# 2. 安装 Apache JMeter

# 2.1 下载

http://jmeter.apache.org/download_jmeter.cgi

For windows10.

# 2.2 开始安装

解压上一步下载的文件。

双击 bin/jmeter.bat 文件。

# 3. 制作 Test Plan

创建一个 Test Plan，使其从 JMeter 发送请求到 web 服务。

默认情况下，Test Plan 已经准备好了，我们只需要设置它。

# 3.1 创建 Thread Group

添加一个 Thread Group，用于处理组的线程。

右击 Test Plan，选择 Add> Threads> Thread Group。

下一步，设置刚才添加的线程的属性。

暂时将 Number of Threads, Ramp-Up period, and Loop Count 都设置为 1，以确认是否能正常连接到目标 Web 系统。

各项属性详细：

key description
Number of Threads(users) 并行执行的线程数
Ramp-up period(seconds) 给定 n 秒，并行处理完上述线程
Loop Count 用上述配置，重复 n 次

# 3.2 添加 Sampler

现在，我们将使用 HTTP request 采样器（sampler）对 Web 服务请求进行性能测试。

右击创建的 Thread Group 并且选择 Add> Sampler> HTTP Request。

然后设置以下请求参数。

# 3.3 添加 Listener

添加一个 Listener 去检查处理结果。

右击 HTTP Request 并且选择 Add> Listener> View Results Tree。

监听器（Listener）有很多种，用于不同的目的。由于此时我们检查的是连接状态，所以我们将选择使用 Tree。

# 3.4 执行并收集结果

运行你创建的 Test Plan。

按下顶部菜单栏的 green triangle button。

这次处理了一次，所以在 Tree 中只显示了一个结果。

如果 HTTP 响应状态码正常，则以绿色标记显示。

打开右侧面板的 Response Data，查看响应内容，如果没有问题，说明该请求已经正常处理。

# 4. 性能测试 - Performance Test

执行测试计划，根据自己的非功能需求改变线程属性的值，在 Listener 中添加一个新的 Aggregate Report。

右击 HTTP Request 并且选择 Add> Listener> Aggregate Report。

按下顶部菜单栏的 green triangle button。

以上图片显示：10 个人以 3 秒间隔分别访问 web 页面 2 次。

测试结果各项指标如下：

header description
Label 采样器标签
# Samples 采样次数
Average 平均响应时间 （ms）。
Median 中位数响应时间 （ms）。
n Line n% 的请求响应时间数（ms）。
Min 最小响应时间（ms）。
Maximum 最大响应时间（ms）。
Error % 请求数的错误率。
Throughput 吞吐量
Received KB/sec 每秒接受的吞吐量（KB）。
Sent KB/sec 每秒传送的吞吐量（KB）。

如果你想一个一个的检查处理结果，使用 View Results in Table 监听器。

右击 HTTP Request 并且选择 Add> Listener> View Results in Table。

测试结果各项指标如下：

header description
Sample # 请求序号。
Start Time 请求开始时间。
Thread Name 线程名称。
Label 采样器标签。
Sample Time(ms) 从发出请求到收到答复的时间。
Status 响应状态。
Bytes 接收数据的字节数。
Sent Bytes 传输数据的字节数。
Latency Time from sending a request to receiving a response.
Connect Time(ms) 连接到服务器的时间。

设置最大并发数，通过无限发送请求和处理查看体验响应速度。

# 5. 其他

还有很多其他的 Listeners。并且你可以记录屏幕操作，然后进行复杂的测试。
