一句话介绍各设计模式。

# 创建型模式

这类模式提供创建对象的机制。能够提升代码的灵活性和可复用性。

  • 工厂方法：由子类决定要创建的具体类是哪一个。
  • 抽象工厂：允许客户创建对象的家族，而无需指定他们的具体类。
  • 单件：确保有且只有一个对象被创建。

# 结构型模式

这类模式将对象和类组装成较大的结构。能够提升结构的灵活和高效。

  • 适配器：封装对象，并提供不同的接口。
  • 组合：客户用一致方式处理对象集合和单个对象。
  • 装饰者：包装一个对象，以提供新的行为。
  • 外观：简化一群类的接口。
  • 代理：包装对象，以控制对目标对象的访问。

# 行为型模式

这类模式负责对象之间的高效沟通和职责委派。

  • 命令：封装请求成为对象。
  • 迭代器：在对象的集合之中游走，而不暴露集合的实现。
  • 观察者：让对象能够在状态改变时被通知。
  • 状态：封装了基于状态的行为，并使用委托在行为之间切换。
  • 策略：封装可以互换的行为，并使用委托来决定要使用哪一个。
  • 模板方法：由子类决定如何实现一个算法中的步骤。
